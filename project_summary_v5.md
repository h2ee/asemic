## 1. 프로젝트 개요

### 핵심 개념
- **긴장(Tension):** 가사를 몰라도 노래에 감동받고, 말 못하는 존재에게도 무언가를 전하고 싶어진다. 소통에서 내용이 차지하는 자리는 우리가 생각한 것보다 작을 수 있다.
- **방법:** 한글 자모에 숫자 규칙을 대응시켜, 관람객이 입력한 텍스트를 언어적 의미가 제거된 아세믹 형태로 변환한다.
- **아세믹 글쓰기 차용 이유:** 의미는 없지만 글의 형식을 유지하기 때문에, 결과물이 소통/연결 의지의 시각화가 된다.

### 세계관 설정
- **나 (관람객) = 외계인** — 인간/비인간, 생물/무생물의 구분이 흐릿한 종족. 지구 방문, 교류/연결을 에너지원으로 삼음. 키보드로 한글 메시지 입력.
- **설치물 = 에너지 수집 기계** — 한글 입력을 받아 아세믹 형태로 출력. 상대의 언어를 아세믹으로 번역하는 도구.

### 세부 시나리오
1. 관람객이 화면 앞에서 하고 싶은 말을 자유롭게 입력. 입력창 주변에 수신자 배치
2. 텍스트가 규칙을 통과해 아세믹 형태로 변환됨. 관람객이 형태를 조절하고 수신자 선택 후 전송
3. 선택한 존재로부터 읽을 수 없는 형태의 답신이 영수증으로 출력됨. 이전 체험자 메시지가 화면에 누적

### 전시 계획
- 5가지 정사각 모니터 사용 (각 모니터당 수신자 1명 + 나머지 1개는 아카이빙)
- 각 모니터 아래에 반투명 흰색 판 — 뒤에서 프로젝터로 비춰 메시지가 판에 나타남
- 전시 기기: Mac(M3) 또는 Windows 11 노트북으로 실행, 모니터에 송출

### main.js 핵심 변경사항

- URL 파라미터 `?receiver=sora` 로 초기 수신자 선택 가능

## 3. 수신자별 시각화

### 외계인 (alien.js) — Three.js 레이마칭
- 3D 구면 사이클로이드, 행성궤도식 에피사이클
- PATH_MODE 7(orbital) 기본값, 8(fm) 선택 가능
- 재질: 홀로그램/투명 유리 느낌

### 소라고둥 (sora.js) — 클라드니 SDF ← 현재 구현 중

#### 글자 생성 방식
각 음절이 **주기적 SDF 필드**:
```glsl
// 클라드니 극좌표 공식
float A = cos(m * theta) * cos(n * PI * r);
float B = cos(n * theta) * cos(m * PI * r);
float d = abs(A - B);  // 0=마디선, 양수=마디선에서 멀어지는 거리값
```
반환값이 일반 SDF의 거리값과 동일한 구조 → IQ의 `smin()`으로 합산 가능.

모든 음절 필드를 `smin()`으로 합산:
- 같은 단어: 음절 간격 ≤ sminK → 마디선이 메타볼처럼 이어짐
- 다른 단어: 음절 간격 > sminK → 자동으로 끊김 (별도 wordId 차단 로직 불필요)

렌더링: `fwidth(d)` 기반 안티앨리어싱 등고선 (IQ 방식)

#### 자모 값 → 파라미터 매핑

**모음 (jung)**
| 자모값 | 변환 | 용도 |
|---|---|---|
| F1 Hz (x) | `f1ToM()` → m (1~M_MAX) | θ방향 모드 수 (꽃잎 수 결정) |
| F2 Hz (y) | `f2ToN()` → n (1~N_MAX) | r방향 모드 수 (동심 링 수 결정) |
| F3 Hz (z) | 미사용 | — |

**초성 (cho)**
| 자모값 | 용도 |
|---|---|
| 조음위치 (x) | f1ToM에서 m에 ±0.3 편이 + darkToColor에서 warm↔cool 색조 |
| 조음방법 (y) | 미사용 |
| 긴장도 (z) | 파도 진폭 간접 영향 (직접 매핑 아님) |

**종성 (jong)** — 현재 완전 미사용

#### 레이아웃 계산 (calcLayout)
```js
const step    = Math.min(0.16, 0.80 / count);
const sminK   = step * 2.0;        // 블렌딩 반경
const wordGap = sminK * 1.02;      // 단어 간격 > sminK → 자동 끊김
const radius  = step * 0.9;        // 음절 하나의 영향 반경
```

#### 복잡도 조절 파라미터
```js
const M_MAX = 4.0;  // f1ToM 상한 (낮출수록 단순, 이전: 7.0)
const N_MAX = 4.0;  // f2ToN 상한
```
m/n 범위 가이드: 2=매우 단순, 3=단순, 4=현재값, 5~7=이전 복잡도

#### 기술 스택
- WebGL2 / GLSL ES 3.0 (변수 인덱스 배열 접근 가능)
- IQ smin (quadratic): `k *= 4.0; float h = max(k-abs(a-b),0.0); return min(a,b)-h*h*0.25/k`
- 알파 블렌딩: `gl.enable(gl.BLEND); gl.blendFunc(gl.SRC_ALPHA, gl.ONE_MINUS_SRC_ALPHA)`
- presence: 가장 가까운 음절까지 거리로 반경 밖 투명 처리

#### sora.js 구현 변천 과정 (참고용)
1. 단순 클라드니 fragment shader
2. 그리드 셀 방식 (각 음절이 격자 칸 하나)
3. Voronoi 방식 (nearest 셀 + 경계 blend) → 선 겹침 문제
4. **현재: 클라드니 = SDF + smin 합산** (선이 실제로 만나서 이어짐)

## 4. 미구현 과제 (sora.js)

### 열감지 컬러
`d` 값을 컬러맵 인덱스로 사용:
```glsl
vec3 heatmap(float t, float choX) {
  // t=0: 마디선(진함), t=1: 배경(흰색)
  // choX로 파랑계열 ↔ 빨강/노랑계열 기울임
  vec3 c0 = vec3(0.0, 0.0, 0.5);   // 파랑 (마디선 중심)
  vec3 c1 = vec3(0.8, 0.0, 0.0);   // 빨강
  vec3 c2 = vec3(1.0, 0.6, 0.0);   // 주황
  vec3 c3 = vec3(1.0, 1.0, 0.85);  // 연노랑
  vec3 c4 = vec3(1.0, 1.0, 1.0);   // 흰 배경
}
```
참고 이미지: 열감지 센서 느낌, 흰→노랑→주황→빨강→파랑 등온선

### TD Displace TOP 방식 부서지는 파도 가장자리
`sylSDF()` 안에서 외곽부(r > 0.6)에만 UV displacement 적용:
```glsl
float displaceStr = smoothstep(0.6, 1.0, r);
float noise = sin(theta * 7.0 + u_time * 1.2)
            * cos(theta * 3.0 - u_time * 0.7)
            * 0.15 * displaceStr;
float rD     = r + noise;
float thetaD = theta + noise * 0.5;
// rD, thetaD로 클라드니 계산
```
- 중심부 패턴은 고정, 외곽만 흔들림
- `waveT`로 진폭 감쇠 → 입력 시 출렁이다 잦아듦
- 종성 연결 후보: 조음위치(x) → 노이즈 주파수, 긴장도(z) → 진폭

### 종성 활용
현재 완전 미사용. 연결 후보:
- 파도 displacement 진폭/주파수
- sminK 조절 (종성 있으면 더 강하게 이어짐)
- 패턴 회전각 오프셋

## 5. 기타 수신자 (미구현)

### 인간 (human) — 에피사이클 2D
- 에피사이클 궤적 위에 정사각형/실루엣 이미지 스탬프
- 궤도 위 스탬프 간격 = 자음 VOT, 크기 = 모음 F1
- 재질: 흑백 2톤, 활판인쇄/고무도장 질감, multiply blend

### 신호등 (signal) — 격자 세포 오토마타
- 자음 긴장도(z) → 색 배정: 빨강(0.67~1.0)/노랑(0.33~0.67)/초록(0~0.33)
- Conway's Life 변형: VOT가 전이 확률 가중

### 민들레 (dandelion) — L-시스템 분기
- 분기 각도 = 자음 조음위치(x) × 40° + 10°
- 세대 수 = round(모음 F1 × 2) + 2
- 애니메이션: 세대별 순차 전개 → 완성 후 홀씨 바람에 날아감

## 7. 폐기/변경된 아이디어

- **사인파 궤적 기반 소라고둥** — 화면 횡단 구조라 부적합. 클라드니로 교체
- **Canvas 픽셀 루프 클라드니** — 매 프레임 순회로 느림. WebGL 셰이더로 교체
- **WebGL1 클라드니** — 배열 변수 인덱싱 불가(GLSL ES 1.0 제한). WebGL2로 전환
- **그리드 셀 방식** — 셀 경계가 인식됨, 선이 합쳐지지 않음. Voronoi → SDF+smin으로 교체
- **Voronoi + 경계 blend** — nearest 셀 선택 후 경계 blend 시도했으나 선 겹침 문제. SDF+smin으로 교체
- **가중평균 dark 합산** — 패턴이 흐릿하게 섞임. soft-max → 최종적으로 smin으로 교체
- **정규화값(0~1) 모음 데이터** — fn≈0 문제로 Hz 원본값으로 교체
